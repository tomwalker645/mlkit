# הפעלה מהירה - נוכחות פלוגתית MVP

קובץ האפליקציה:
- `C:\Users\User\Documents\נוכחות\נוכחות_פלוגתית_MVP.html`

## מה זה נותן כרגע
- התחברות משתמשים (אימייל/סיסמה)
- תפקידים:
  - מפקדת פלוגה
  - מפקדת מחלקה
  - צופה מורשה
- מבנה נתונים: פלוגה > מחלקות > צוותים > חיילים
- דיווח נוכחות מחלקתי (V/X/0/? לכל חייל)
- דשבורד דוחות לכל הפלוגה למי שיש הרשאה
- ניהול הרשאות צפייה פלוגתית ע"י מפקדת פלוגה

## שלב 1: פתיחת פרויקט Firebase
1. פתח Firebase Console.
2. צור פרויקט חדש.
3. הפעל Authentication -> Email/Password.
4. הפעל Firestore (Production או Test לפי צורך).
5. צור Web App וקבל את פרטי `firebaseConfig`.

## שלב 2: הדבקת firebaseConfig
בקובץ `נוכחות_פלוגתית_MVP.html` חפש את האובייקט:
`const firebaseConfig = { ... }`
והחלף את כל ערכי `PUT_YOUR_...` בערכים האמיתיים.

## שלב 3: אינדקסים ב-Firestore
האפליקציה משתמשת בשאילתות משולבות. אם Firestore יבקש אינדקס - צור לפי הקישור ש-Firebase מציג.

## שלב 4: כללי אבטחה (חשוב)
כרגע ההרשאות נאכפות גם בצד לקוח. חובה להגדיר Firestore Security Rules שמאכפים:
- גישה רק למשתמש מחובר
- גישה רק למסמכים של `companyCode` של המשתמש
- רק `company_commander` יכולה לשנות `canViewAll`
- `platoon_commander` יכולה ליצור דוחות רק למחלקה שלה

## שלב 5: הרצה
אפשר לפתוח את ה-HTML בדפדפן אחרי שמילאת `firebaseConfig`.
מומלץ להריץ דרך שרת סטטי קטן (כדי למנוע מגבלות דפדפן מסוימות):
- VS Code Live Server, או
- כל שרת HTTP מקומי.

## הערה חשובה על MVP
זהו בסיס עובד להיררכיה פלוגתית והרשאות.
לשימוש מבצעי מומלץ להוסיף:
- Firestore Rules מלאים
- Audit Log ייעודי
- גיבוי/ייצוא דוחות מתקדם
- בדיקות עומס והרשאות
